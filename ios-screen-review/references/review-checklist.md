# iOS Screen Review Checklist

## Purpose and Hierarchy
- Primary action is clear and reachable.
- Visual hierarchy separates primary, secondary, and tertiary content.
- Screen title matches the user goal.

## Layout and Spacing
- Consistent spacing rhythm across sections.
- Alignment is consistent within groups.
- Content is not crowded; density matches task frequency.

## Navigation and Affordances
- Navigation affordances are visible and predictable.
- Buttons look tappable and have labels.
- Destructive actions are clearly marked and separated.

## Content and Copy
- Labels are concise and unambiguous.
- Error and empty states are helpful and actionable.
- No redundant text or duplicated headings.

## States
- Loading state is present and does not block unnecessarily.
- Empty state explains why and offers next steps.
- Error state provides recovery or retry.
- Offline behavior is defined when relevant.

## Accessibility
- Touch targets meet 44x44pt minimum.
- Text scales with Dynamic Type.
- Icon-only controls have labels.

## Visual Polish
- Colors respect contrast and system semantics.
- Icons match platform style and weight.
- Rounded corners use consistent radius.
